Methodology

  • Job filter: DEL-suffix jobs only (ucjbNumber ending in 'DEL'), Speed ID 95, JobRelationshipTypeID = 20 (child), client codes WOOP/WOOPH/WOOPW/WOOPC, non-void.
  • Run grouping: Unique (runname + date) combinations. Run names repeat weekly.
  • DEL-only revenue: Sum of ucjbAmount for DEL children in the run — this is the correct MFV base.
  • MFV received: Sum of fuelsurchargeamount per job (actual MFV paid from tblFuelSurcharge rates).
  • Driven KM: Nearest-neighbour TSP from depot through all delivery points and back, × 1.27 road multiplier.
  • Extra fuel cost: (Pump $3.70 − Base $1.84) × 13L/100km × driven KM. Only the cost above the $1.84/L baseline — the MFV only needs to cover excess fuel cost.
  • Surplus/deficit: MFV received minus extra fuel cost. Positive = over-compensated, negative = under-compensated.
  • Fair MFV %: The percentage of DEL revenue that would exactly cover extra fuel cost at these parameters.
  • Pre-March 2025: Old billing model (rel=1, no suffixes) — each job was standalone. Only runs with non-zero fuelsurchargeamount included.
  • Post-March 2025: New parent-child model — DEL suffix = delivery leg, LH/LHP = linehaul. Only DEL children used for MFV base calculation.
